# AI Coding 开发规范

## 1. 概述

本规范旨在指导团队在使用AI辅助工具进行代码开发时的最佳实践，确保代码质量、安全性和可维护性。

## 2. 代码风格规范

### 2.1 命名规范
- **变量名**：使用小驼峰命名法（camelCase）
- **函数名**：使用小驼峰命名法（camelCase）
- **类名**：使用大驼峰命名法（PascalCase）
- **常量**：使用全大写字母，单词间用下划线分隔
- **文件/目录名**：使用小写字母，单词间用连字符（-）分隔

### 2.2 代码格式
- 使用 2 个空格进行缩进
- 行尾不添加空格
- 每行代码长度不超过 80 个字符
- 函数定义与调用时，参数换行应保持一致的缩进

### 2.3 注释规范
- 类和函数必须添加 JSDoc 风格的注释
- 复杂逻辑必须添加行内注释
- 注释应使用中文，清晰明了地解释代码意图

## 3. AI 工具使用规范

### 3.1 工具选择
- 推荐使用：GitHub Copilot、ChatGPT Code Interpreter
- 禁止使用：未经验证的第三方AI工具

### 3.2 提示词（Prompt）规范
- 明确描述需求和约束条件
- 提供足够的上下文信息
- 指定期望的代码风格和格式
- 要求生成的代码包含适当的注释

### 3.3 代码审查
- 所有AI生成的代码必须经过人工审查
- 检查代码的安全性、性能和可读性
- 确保代码符合项目的编码规范

## 4. 安全规范

### 4.1 数据安全
- 禁止在提示词中包含敏感信息（如API密钥、密码等）
- 避免将包含敏感数据的代码提交到版本控制系统

### 4.2 代码安全
- 定期使用静态代码分析工具检查代码安全
- 注意防范常见的安全漏洞（如SQL注入、XSS攻击等）
- 遵循最小权限原则

## 5. 性能优化

### 5.1 代码优化
- 避免不必要的计算和循环
- 合理使用缓存机制
- 优化数据库查询

### 5.2 资源管理
- 及时释放不再使用的资源
- 合理使用内存和CPU资源
- 避免内存泄漏

## 6. 版本控制

### 6.1 Git 规范
- 提交信息应清晰明了，遵循约定式提交规范
- 分支命名应反映功能或修复内容
- 定期进行代码合并和冲突解决

### 6.2 代码审查
- 所有代码变更必须经过代码审查
- 审查重点包括代码质量、安全性和性能
- 提供具体的反馈和改进建议

## 7. 测试规范

### 7.1 测试策略
- 为核心功能编写单元测试
- 进行集成测试确保模块间的协作正常
- 编写端到端测试验证整个系统的功能

### 7.2 测试覆盖率
- 代码测试覆盖率应不低于 80%
- 重点测试边界情况和异常处理
- 定期运行测试套件确保代码质量

## 8. 文档规范

### 8.1 代码文档
- 为公共API编写详细的文档
- 文档应包含使用示例和参数说明
- 定期更新文档以反映代码变更

### 8.2 项目文档
- 维护项目README文件，包含项目简介、安装步骤和使用说明
- 编写架构文档，描述系统设计和组件关系
- 记录重要的设计决策和技术债务

## 9. 团队协作

### 9.1 沟通规范
- 使用统一的沟通工具和频道
- 及时分享代码变更和技术决策
- 定期举行代码评审会议

### 9.2 知识共享
- 建立知识库，记录常见问题和解决方案
- 分享学习资源和最佳实践
- 鼓励团队成员互相学习和提高

## 10. 持续改进

### 10.1 规范更新
- 定期审查和更新本规范
- 收集团队成员的反馈和建议
- 适应新技术和工具的发展

### 10.2 技能提升
- 鼓励团队成员学习新的编程技术和AI工具
- 组织技术分享和培训活动
- 建立学习型团队文化
