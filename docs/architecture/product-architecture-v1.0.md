# PawMind 产品架构文档 v1.0

> 从 PRD v1.0 中提取的产品架构定义，描述系统整体结构、信息层级和核心数据模型。

## 产品定位

PawMind 是面向**城市养宠人群**的 AI 宠物陪伴 App，通过 AI 语音互动、行为分析和健康追踪，帮助主人在工作繁忙或出差期间仍能与宠物保持情感连接，并及时发现健康隐患。

**核心理念**：宠物不会说话，但 AI 可以帮它表达。

## 核心数据流

```
宠物行为/健康数据 → AI 分析引擎 → 情绪状态评估 → 陪伴互动 / 健康预警 → 主人感知 & 行动
```

## 信息层级

```
用户账号（全局唯一，支持多宠物管理）
  └ 宠物档案（每只宠物独立档案）
      ├ 基础信息（品种 / 年龄 / 体重 / 疫苗记录）
      ├ AI 陪伴记录（对话历史 / 声音互动记录）
      ├ 健康数据（每日体征 / 异常预警 / 就医记录）
      ├ 成长日记（照片 / 视频 / 打卡里程碑）
      └ 行为分析（活跃度趋势 / 情绪曲线 / 饮食记录）
```

## 核心数据模型

```
用户（User）
  ├ id, email, password_hash, nickname, avatar
  └ created_at, updated_at

宠物档案（PetProfile）（每用户可创建多只，按 pet_id 唯一）
  ├ 元数据（name / species / breed / birthday / avatar）
  ├ 健康基线（weight / normal_heart_rate / vaccine_list）
  ├ 关联关系（属于 user_id / 可被多设备访问）
  └ 状态信息（active / archived）

AI 对话记录（AIConversation）
  ├ session_id / pet_id / created_at
  ├ 消息列表（role: user|ai, content, timestamp）
  └ 情绪标签（joy / calm / anxious / excited）

健康日志（HealthLog）
  ├ log_id / pet_id / date
  ├ 体征数据（weight / appetite / activity_level）
  ├ 异常标记（is_alert / alert_type / severity）
  └ 关联就医记录（vet_visit_id）

成长记录（GrowthRecord）
  ├ record_id / pet_id / content_type
  ├ 媒体（media_url / description / tags）
  └ created_at

关联关系：
  - 用户 1 对多 宠物档案
  - 宠物档案 1 对多 AI 对话记录
  - 宠物档案 1 对多 健康日志
  - 宠物档案 1 对多 成长记录
  - 健康日志 → 删除宠物档案时级联归档（不物理删除）
```

## AI 能力架构

```
触点 1：宠物视角对话（对话级）
  - 能做：以宠物第一人称回应，生成符合宠物性格的对话
  - 上下文：宠物档案 + 今日状态 + 历史对话摘要
  - 记忆：保留近 10 轮对话；跨天自动生成摘要保留长期记忆

触点 2：健康异常检测（模块级）
  - 能做：检测体征异常、判断严重程度、给出行动建议
  - 上下文：宠物健康基线 + 近 30 天健康数据 + 品种特征库
  - 记忆：无状态分析；历史预警存储在健康日志中

触点 3：成长报告生成（对象级）
  - 能做：读取月度数据，生成有温度的图文总结
  - 上下文：当月所有成长记录 + 健康数据 + 里程碑事件
  - 记忆：无状态生成，报告生成后独立存储
```

## 设计原则

| 原则 | 说明 |
|------|------|
| **情感优先于功能** | AI 回复宁可短一些、有趣一些，也不要长篇大论但冷冰冰 |
| **主人不应做判断员** | 健康数据永远给出"下一步行动"，不把原始数据扔给用户 |
| **宠物是主角** | 所有内容的叙述主体是宠物，AI 帮宠物"发声" |
| **陪伴轻触化** | 每次互动 30 秒内完成，不设繁重打卡任务 |
| **降级而非崩溃** | AI 失败时保留基础功能，不因 AI 服务不可用让 App 无法使用 |

## 版本演进

| 版本 | 目标 |
